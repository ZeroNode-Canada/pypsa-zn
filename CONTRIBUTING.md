# Contributing to ZeroNode
 
Thank you for your interest in contributing to ZeroNode. We welcome contributions
from the community and appreciate your effort to improve the project.
 
## How to Contribute
 
1. **Fork the Repository.** Create a personal fork of the repository on GitHub.
 
2. **Create a Branch.** Make your changes in a new branch from `main`.
 
3. **Follow Code Standards.** Ensure your code follows the existing code style and
includes appropriate SPDX license headers (see below).
 
4. **Test Your Changes.** Run the existing test suite and add tests for new
functionality where appropriate.
 
5. **Submit a Pull Request.** Open a pull request against the `main` branch with
a clear description of the changes and the problem they solve.

## Contributor License Agreement
 
Before your first contribution can be merged, you must agree to the ZeroNode
Contributor License Agreement (CLA). The CLA ensures that you have the right to
contribute the code and that ZeroNode can distribute it under Apache 2.0.
 
Please review the CLA at [CLA.md](CLA.md). To indicate your agreement, include
the following statement in your first pull request:
 
I have read and agree to the ZeroNode Contributor License Agreement.
 
## Dependency Policy
 
Do not introduce new dependencies without verifying that they are licensed under
a permissive license (MIT, BSD, Apache-2.0). Dependencies licensed under GPL,
AGPL, proprietary, or academic-use-only licenses are not permitted. If you are
unsure about a dependency's compatibility, please raise the question in your pull
request.
 
## Code of Conduct
 
All contributors are expected to adhere to professional standards of conduct.
Disrespectful or inappropriate behavior will not be tolerated.
